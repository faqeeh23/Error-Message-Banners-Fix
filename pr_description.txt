### Why is this Change Important & Necessary?
Closes #12431

Submitty relies on message banners (Warnings, Errors, Success, Info) to relay critical information. Historically, these messages lacked a standardized style, often relying purely on colored containers with inconsistent typography and contrast globally. This change introduces a reliable, accessible, and easily reusable architecture for all system notifications.

### What is the New Behavior?
This PR introduces a highly reusable Vue.js component (`SubmittyAlert.vue`) and integrates it closely with Submitty's global stylesheets.

- **Componentized & Content-Agnostic**: It uses a standard `<slot>` wrapper allowing it to support any dynamic content while enforcing standard UI (via the `type` prop).
- **Submitty Setup**: All structural classes have been moved directly to `server.css`, and semantic theme values have been mapped directly to `colors.css`, fulfilling the issue's structural request and supporting instant Light/Dark mode toggling.
- **Colorblind-Safe Accessibility**: Colors are strictly utilizing the Okabe-Ito colorblind-safe palette (e.g., Deep dark reds for errors, high-contrast golden-yellows for warnings) rather than default hex values.
- **Explicit Labels & Screen Readers**: Banners no longer force users to rely solely on color. The component integrates `aria-live="polite"`, `sr-only` labels, and dynamic high-contrast Icon topography.
